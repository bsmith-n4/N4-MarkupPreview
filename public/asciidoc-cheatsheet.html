9. Tables

9.1. Basics

Tables begin and end with |=== and a vertical bar | will create a new cell. Below we have the minimum required to produce a basic table:

Example: 
|===
| Hello | world
|===
Output:

Hello

world

The number of cells in the first line will dictate how many colmuns the table will have:

Example: 
|===
| Fruit | Veg | Nuts
| peaches
| broccoli | pecan | apples | beetroot | cashew
|===
Output:

Fruit

Veg

Nuts

peaches

broccoli

pecan

apples

beetroot

cashew

Leaving an empty line after the first row will format that row as a header (centre-aligned and in bold). The table will inherit the number of columns defined in the header row. Empty lines after the header are ignored - new lines do not indicate a new row:

Example: 
|===
| Backend | Description

| html (or html5) | HTML5, styled with CSS3 (default). | pdf

| PDF, a portable document format.
|===
Output:
Backend Description

html (or html5)

HTML5, styled with CSS3 (default).

pdf

PDF, a portable document format.

9.2. Table Attributes

Attribute Description

[options="footer"]

The last row of a table is styled as a footer

[width=n]

Overall table width spans n% page width

[frame=topbot]

No border is drawn on the left and right sides of the table.

[frame=sides]

No border is drawn on the top and bottom of the table.

[frame=none]

No border is drawn around the table.

[grid=rows]

No border is drawn between columns in the table.

[grid=cols]

No border is drawn between rown in the table.

[grid=none]

No borders are drawn between rows and columns in the table.

[caption="Table A. "]

A table with a title will be defined as a formal table and will be rendered with an enumerated caption. This attribute replaces Table 1. etc. with custom caption.

9.5. Column Styles

Styles can be applied with shorthand before the | vertical bar that creates a cell as follows:

Style Value Description

AsciiDoc

a

Any block-level elements (paragraphs, delimited blocks and block macros) may be contained within the column. The elements will be processed and rendered.

Emphasis

e

Text is italicized

Header

h

Header styles are applied to the column

Literal

l

Column content is treated as if it were inside a literal block

Monospaced

m

Text is rendered in monospaced font

None (default style)

d

Text is handled like a normal paragraph. Supports all markup (i.e., inline formatting, inline macros) that is permitted in a paragraph.

Strong

s

Text is bolded

Verse

v

Column content is treated as if it were inside a verse block

We can set default formatting of columns as in the block declaration [cols="h,m,s,e"]. This creates a table with four columns with the formatting of header, monospace, strong, and emphasis respectively. This formatting can be overridden in individial cells by indicating with shorthand before the cell below:

Example: 
[cols="h,m,s,e"]
|===
| heading | monospace | strong(bold) | emphasis(italic)

4+a|

==== AsciiDoc
Content formatted as AsciiDoc.

* Including lists or other blocks
** Like a quote block:
[quote]
Don't quote me on this, but...

| content | here | retains| formatting
|===
Output:

heading

monospace

strong(bold)

emphasis(italic)
