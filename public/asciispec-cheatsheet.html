2. Examples

2.1. Definition Block

Source

.Definition Site Structural Typing
[def]
--
If a type T is declared as structural at its definition, T.defStructural is true.

  1. The structurally defined type cannot be used on the right hand side of the instanceof

  2. A type X is a subtype of a structurally defined type T…​

Furthermore…​
--

Output

Definition: Definition Site Structural Typing

If a type T is declared as structural at its definition, T.defStructural is true.

  1. The structurally defined type cannot be used on the right hand side of the instanceof

  2. A type X is a subtype of a structurally defined type T…​

Furthermore…​

2.2. Inline BibTeX Macro

Source

Two of my favourite books are cite:[ECMA15a,Canning89a] the ECMAScript Language Specification and F-bounded polymorphism.

Thanks for reading, be sure to check my biblio:

bibliography::[]

Output

Two of my favourite books are [ECMA15a; Canning89a] the ECMAScript Language Specification and F-bounded polymorphism.

Thanks for reading, be sure to check my biblio:

ECMA. (2015). ECMAScript 2015 Language Specification. Retrieved from http://www.ecma-international.org/publications/files/ECMA-ST/Ecma-262.pdf

Canning, Peter and Cook, William and Hill, Walter and Olthoff, Walter and Mitchell, John C.. (1989). F-bounded Polymorphism for Object-oriented Programming. Retrieved from http://doi.acm.org/10.1145/99370.99392
Configuration:

:bib-file: path/to/bibtexfile.bib

2.4. Inline Task Macro

Source

AsciiDoc was decided as a suitable syntax for documentation. task:IDE-2273[]

A bug has already task:GH-135[] been filed…​

Output

AsciiDoc was decided as a suitable syntax for documentation testing.

A bug has already been filed…​
IDE-2273 GH-135
Config.adoc Example: 
:task_def_GH-: GitHub;IDE Bugs;https://github.com/NumberFour/N4JS/issues/${TASK_ID};icons/github.png;GH-${TASK_ID}
:task_def_IDE-: Jira;IDE Backlog;https://jira.numberfour.eu/browse/IDE-${TASK_ID};icons/jira.png;IDE-${TASK_ID}
